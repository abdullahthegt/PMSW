📈 DATA GENERATION PARAMETERS

Team Generation:
├─ Size: 5-20 members (configurable)
├─ Roles: Dev 50%, Tester 30%, Architect 10%, PO 5%, Safety 5%
├─ Seniority: Junior 30%, Mid 50%, Senior 20%
├─ Efficiency: N(1.0, 0.1) - higher for seniors
└─ Availability: N(35, 5) hours/week

Backlog Generation:
├─ Size: 15-50 stories (configurable)
├─ Types: Requirement, Design, Code, Test, Safety, Integration
├─ Story Points: Gamma distribution (1,2,3,5,8,13,21,34)
├─ ASIL: QM 40%, A 20%, B 20%, C 15%, D 5%
└─ Dependencies: V-Model structure (Design→Req, Code→Design, etc.)

Historical Sprints:
├─ Count: 5-20 sprints (configurable)
├─ Velocity: Trending (±2 SP/sprint)
├─ Risk Events: 20% probability
└─ Variance: ±10% normal variation

🧮 ALGORITHMS & FORMULAS

VELOCITY FORECASTING:
V_sprint ~ N(μ, σ²)
where μ = rolling_avg(completed_sp, lookback_sprints)
      σ = std_dev(completed_sp, lookback_sprints)

Confidence Interval:
lower_bound = μ + trend - z * σ
upper_bound = μ + trend + z * σ
where z = 1.645 for 90%, 0.842 for 80%

Success Probability:
P(success) = P(V_sprint ≤ capacity)
           = 1 - CDF_normal(capacity, μ, σ)

RESOURCE ALLOCATION:
Priority Score = (ASIL_weight * story_points) / estimated_hours + priority_bonus
Assignment: Greedy best-fit (max remaining capacity)
Feasibility: Total effort ≤ team capacity

📌 KEY TAKEAWAYS

1. SYNTHETIC DATA
   Problem: No public automotive project data
   Solution: Generate realistic synthetic datasets
   Benefit: Test DSS without proprietary info

2. PROBABILISTIC VELOCITY
   Problem: Single-point estimates fail (Flaw of Averages)
   Solution: Model velocity as probability distribution
   Benefit: Risk-aware planning with confidence intervals

3. SKILL-BASED ALLOCATION
   Problem: Resources aren't fungible in automotive
   Solution: Match tasks to required skills with heuristics
   Benefit: Realistic bottleneck identification

4. INTERACTIVE DECISION SUPPORT
   Problem: Static reports don't support decisions
   Solution: Real-time dashboard with visualizations
   Benefit: Explore scenarios and make informed choices
